Do Docker containers dedicated for compliance components, like monitoring or logging, count against my allocated RAM?

No. All resources required to run Datica’s compliance layer supporting your environment are included with every account. RAM allocated for customers is reserved strictly for your technology, such as application or database hosting. 

Do backups and disaster recovery count against my allocated disk space?

No. Similar to RAM allocation, available disk space is entirely yours. Nightly backups are a basic feature of Datica’s platform, and the space they required is baked into the product.

What is included in the Business Associate Agreement?

Datica’s BAA is publicly available here: policy.datica.com. We encourage you to review it and compare to others. If you have any questions about your coverage, please feel free to contact us. 

Is your Business Associate Agreement negotiable?

Enterprise-tier customers may negotiate separate BAAs. Customers on the developer and growth tiers should plan to use Datica’s standard BAA.

Am I able to pick which cloud infrastructure provider I wish to deploy my application?

This capability is only available to Enterprise-tier customers.

What if I need extra help, like to re-architect my application for containerization, etc?

Datica provides a variety of custom Services that most likely will fit your need.

How does Datica handle licensing fees for technology like SQL Server or Oracle DB via AWS RDS?

Licensing fees are pass through to our customers. If a customer requires proprietary technology within their environment configuration, Datica does not take on the license fees to use that technology.

I want to deploy an application.

Technical Requirements

Datica abstracts away all the things you used to do with on-premise or IaaS providers, like manage web servers, VMs and hardware.

The only thing you need to worry about is how much computing power and data storage resources you need in order to efficiently run your application. You do this by mapping dedicated RAM to individual services. Here’s a typical digital health application stack:

Code service backend, like Java 1x 2GB RAM
Database service, like Postgres 1x 2GB RAM
Redis for caching 1x 1GB RAM
Total required: 5GB RAM

This build fits nicely in the Standard plan. Should you need additional resources for other services as you grow, adding more RAM is as simple as clicking a button. Extra RAM is $100/GB.

Service Requirements

Beyond infrastructure, consider your support and service needs. Tech support, SLAs, and compliance extras are determined by the plan you select.

Start with a free trial of the Standard Plan or choose the Professional Plan

I want to integrate with EHRs

Integration Environment

You can purchase a one-click-install integration environment, which has all the technology components necessary to handle EHR integration: Highly-Available configured Mirth, load balancers, Postgres backups, and VPN management.

The environment takes a minimum of 8GB of RAM but also requires VPNs, which means the Professional Plan is the right place to start.

VPNs

Since each integration requires a VPN into the specific site, you will work with the Datica team to help set up each VPN. We charge a $1,500/year/VPN connection maintenance fee to ensure they are in a 24/7 Keep-Alive status.

Professional Services

Since each integration is its own journey where you integrate into a hospital’s private network, you may need the occasional help. Datica offers wide variety of professional services should you need it.

Per-site go-live management: We take on the entire process of a specific site’s integration from start to finish. $6,000 one-time.

Integration Engineering Assistance: We can help with any extra ad-hoc help with integration tasks at an hourly rate.
